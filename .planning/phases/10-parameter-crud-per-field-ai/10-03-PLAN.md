**Per-field AI on metadata fields (form.go):**

Add AI trigger buttons to each metadata field in the form:
- When a field is focused and AI is available, show a dim "⚡AI" indicator at the right edge of the field.
- User presses a keybinding (e.g., Ctrl+G while in a field) to trigger per-field AI generation.
- The trigger sends a `perFieldAIRequestMsg{field: "description", context: formSnapshot}` where `formSnapshot` captures the current values of all fields as context.
- Requires at least one "seed" field: title, description, or command must have content. If no seed, show an inline warning "Add a title, description, or command first" and don't trigger.

**Per-field AI on parameter fields (param_editor.go):**

Similarly add AI trigger within the expanded param editor:
- When focused on a param's default value, enum options, or dynamic command field, Ctrl+G triggers AI.
- Context includes: workflow name, description, command, param name, param type, and all other param data.
- AI suggests: for default values, a sensible default; for enum options, a list of common options; for dynamic commands, a shell command.

**AI async commands (ai_actions.go):**

Add new message types and commands:

```go
type perFieldAIRequestMsg struct {
	fieldName string   // "name", "description", "command", "tags", "param:{index}:default", "param:{index}:options", "param:{index}:dynamic_cmd"
	context   map[string]string // snapshot of all field values for AI context
}

type perFieldAIResultMsg struct {
	fieldName string
	value     string
	err       error
}
```

Add `perFieldAICmd(fieldName string, context map[string]string) tea.Cmd`:
- Uses `ai.GetGenerator()` to get the generator.
- Calls `gen.Autofill()` with a constructed workflow from the context, requesting only the specific field.
- For param-level fields, construct an `AutofillRequest` with `Fields: ["args"]` and extract the specific param's value from the result.
- Returns `perFieldAIResultMsg`.

**Ghost text rendering:**
- When a `perFieldAIResultMsg` arrives for a field, store the suggestion in a `ghostText map[string]string` on FormModel.
- Render the ghost text as dimmed text appended to the field's current value (or as placeholder if field is empty).
- Use the theme's `Colors.Dim` for ghost text color.
- Enter on a field with ghost text accepts the suggestion (sets the field value to the suggestion).
- Esc on a field with ghost text dismisses it (clears the ghost text, field value unchanged).
- Any keystroke (typing) clears the ghost text for that field.

**Loading indicator (PFAI-02):**
- When per-field AI is in progress, show a spinner/loading indicator on that specific field.
- Use `fieldLoading map[string]bool` on FormModel to track which fields are loading.
- Render: replace the "⚡AI" indicator with "⣾" (spinning braille character — static is fine per Phase 5 decision) while loading.
- Only the target field shows loading; other fields remain editable (per-field is non-blocking per CONTEXT.md).

**Autofill button (form.go):**
- Add an "⚡ Autofill" button rendered below the folder field and above the parameters section.
- Focused with Tab navigation (it's a "field" in the tab order).
- Enter on it triggers autofill — fills all empty fields using AI.
- During Autofill: show per-field loading spinners on each field being generated. Lock the form (ignore keystrokes except Esc to cancel) per CONTEXT.md decision about preventing context mismatch.
- Autofill skips fields that already have user-entered values.
- AI result is applied via `mergeAutofillResult` pattern (existing in ai_actions.go) extended to handle form fields.
- After autofill, each filled field shows the result as ghost text for user to accept/dismiss individually.

**Command-to-params AI (ai_actions.go):**
- Add a "Suggest Parameters" feature: user can trigger analysis of the current command to suggest parameters.
- Define `suggestParamsCmd(command string) tea.Cmd` that calls `ai.GetGenerator()` and asks it to identify parameterizable parts.
- The AI returns suggested `[]store.Arg` entries.
- Suggested params appear in the param editor as ghost entries (dimmed, not yet committed) for user to accept individually.

**model.go changes:**
- Route `perFieldAIResultMsg` to the form when in `viewCreate` or `viewEdit` state.
- Handle ghost text accept/dismiss messages.

**Keybinding:** Use Ctrl+G for per-field AI trigger (reuse the existing `GenerateAI` binding concept, but in-form context). Add `FormAI key.Binding` to keyMap: Ctrl+G — "AI suggest".
