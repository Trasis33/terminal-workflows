  <action>
Create `internal/manage/param_editor.go` with a `ParamEditorModel` Bubble Tea model.

**Data structure:**
- `params []paramEntry` where `paramEntry` holds: `name string`, `paramType string` (text/enum/dynamic/list), `defaultVal string`, `options []string`, `dynamicCmd string`, `description string`, `expanded bool`, `nameInput textinput.Model` (for inline name editing when expanded).
- `cursor int` — which param row is focused.
- `editing bool` — whether the expanded param's name field is in edit mode.
- `theme Theme` for styling.
- `width int` for layout.
- `confirmDelete int` — index of param pending delete confirmation (-1 = none).

**Initialization:**
- `NewParamEditor(args []store.Arg, theme Theme, width int) ParamEditorModel` — converts `[]store.Arg` to `[]paramEntry`.

**Add parameter (PCRD-01):**
- Appends a new `paramEntry{name: "new_param", paramType: "text", expanded: true}` to the list.
- Sets cursor to the new param index.
- Creates a focused `textinput.Model` for the name field with the text "new_param" pre-filled and selected.
- Sets `editing = true` so the user can immediately type the name.
- If a param was previously expanded, collapse it first (accordion: only one expanded at a time).

**Remove parameter (PCRD-02):**
- When user presses delete key on a param row, set `confirmDelete = cursor`.
- Show inline confirmation text: "Delete {name}? y/n" on that row.
- On "y": remove the param from the list, adjust cursor. On "n" or Esc: cancel.
- A workflow can have zero parameters — removing the last param is allowed, showing an empty state.

**Accordion expand/collapse:**
- Enter on a collapsed row expands it (shows all editable fields for that param).
- Only one param can be expanded at a time. Expanding another collapses the current one.
- When expanded, show: name (editable), type (display-only for now — Plan 02 handles type change), default value (display-only for now).

**Navigation:**
- Up/Down (k/j) navigates between param rows when no field is focused.
- Tab moves between fields within an expanded param.

**View rendering:**
- Collapsed row: `  ▸ {name}  ({type})` with cursor highlight on focused row.
- Expanded row: `  ▾ {name}  ({type})` with indented fields below.
- Bottom: `  + Add Parameter` button row (selectable, triggers add on Enter).
- Use theme colors: param name in `Colors.Primary`, type in `Colors.Dim`, focused row with `Colors.Accent` background.
- When confirmDelete is active, that row shows the confirmation prompt in `Colors.Error`.

**Keybindings (add to keys.go):**
- `ParamAdd` — Ctrl+N — "new param"
- `ParamDelete` — Ctrl+D — "delete param"
- Add these to `keyMap` struct and `defaultKeyMap()`. Add to `FullHelp` in the form/editor group.

**Methods:**
- `Init() tea.Cmd`
- `Update(msg tea.Msg) (ParamEditorModel, tea.Cmd)` — handles key events for navigation, add, remove, expand/collapse.
- `View() string` — renders the param list.
- `ToArgs() []store.Arg` — converts current `[]paramEntry` back to `[]store.Arg` for persistence.
- `SetWidth(w int)` — updates layout width.
- `ParamCount() int` — returns number of params.

Do NOT implement type change, metadata editing, or AI features in this task — those come in Plan 02 and Plan 03.
