# wf-workflows skill

Load this skill when the task is about creating, editing, explaining, or reviewing `wf` workflows.

Use it for:

- writing new workflow YAML
- converting an idea into a usable `wf` workflow
- choosing between `text`, `enum`, `dynamic`, and `list`
- explaining how `list_cmd` works
- producing realistic Git, Bitbucket, or Azure workflow examples

Do not use it as the primary skill for debugging broken workflows.

For debugging and diagnosis, use:

- `../wf-workflow-troubleshooting/SKILL.md`

Quick rule:

- building or explaining -> `wf-workflows`
- diagnosing or fixing -> `wf-workflow-troubleshooting`
